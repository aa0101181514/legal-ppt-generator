# 引用格式與法律偵探 API 用法

本文件為本專案的引用規則權威來源，所有判決 / 卷證 / 法條之引用格式皆以本文件為準。

---

## 一、判決字號書寫格式

### 標準格式（投影片可見文字）

| 法院類別 | 格式 | 範例 |
|---|---|---|
| 最高法院 | `最高法院 {年} 年度{字}字第 {號} 號{刑事/民事}判決` | 最高法院 108 年度台上字第 2027 號刑事判決 |
| 最高行政法院 | `最高行政法院 {年} 年度{字}字第 {號} 號判決` | 最高行政法院 107 年度判字第 369 號判決 |
| 高等法院 | `台灣{區}高等法院 {年} 年度{字}字第 {號} 號{類別}判決` | 台灣高雄地方法院 115 年度金重訴字第 1 號刑事判決 |
| 地方法院 | `台灣{區}地方法院 {年} 年度{字}字第 {號} 號{類別}判決` | 台灣台北地方法院 113 年度訴字第 XX 號民事判決 |
| 裁定 | 將「判決」改為「裁定」 | 最高法院 110 年度台抗字第 XX 號裁定 |

**不可出現在投影片的寫法**：「108 台上 2027」「107 判 369」（屬內部溝通用簡寫，正式場合不符體例）。

### 二度以上引用

首次出現用完整格式，之後可用：

- `同前判決`（緊接前一段引用同判決時）
- `最高行 107 判 369 判決`（已在首次出現時建立簡稱後）

## 二、判決驗證 API

### Endpoint

```
GET https://api.dr-lawbot.com/api/search?q={查詢字串}
```

### 查詢字串寫法

以下三種任一皆可，API 會正規化：

1. 完整字號：`最高法院 108 年度台上字第 2027 號`
2. 簡短字號：`108 台上 2027`
3. 關鍵字：`傳聞證據 刑事訴訟法 159 條`（用於探索型搜尋）

### 在對話中呼叫

你（Claude）使用內建 `web_fetch` 工具：

```
web_fetch("https://api.dr-lawbot.com/api/search?q=108 台上 2027")
```

中文與空白 Claude.ai 會自動處理 URL 編碼。

### 回傳 JSON 結構

```json
{
  "page": 1,
  "size": 20,
  "has_next": false,
  "search_mode": "exact_case" 或 "fulltext" 或其他,
  "sort": "relevance" 或 "date_desc",
  "results": [
    {
      "doc_id": "TPSM,108,台上,2027,20191211,1",
      "jyear": "108",
      "jcase": "台上",
      "jno": "2027",
      "jdate": "20191211",
      "title": "違反廢棄物清理法等罪",
      "court_level": "supreme",
      "case_type": "criminal",
      "is_reference": true,
      "reference_type": "reference",
      "main_preview": "…（判決主文、事實、理由節錄）…",
      "snippet": "…（全文搜尋命中段落）…",
      "doc_type_label": "判決"
    }
  ]
}
```

### 欄位用途

| 欄位 | 用於 |
|---|---|
| `doc_id` | 法院代號辨識（前綴） + 建立 dr-lawbot 內部連結（可選） |
| `jyear / jcase / jno` | 驗證字號吻合 |
| `jdate` | 判決日期（YYYYMMDD） |
| `title` | 判決案由，可於首次引用後補充說明 |
| `court_level` | supreme / high / district |
| `case_type` | criminal / civil / administrative |
| `main_preview` | **可作 blockquote 引文來源** |
| `snippet` | **可作 blockquote 引文來源**（全文搜尋命中段落） |
| `doc_type_label` | 「判決」或「裁定」，用於書寫時選字 |

## 四、引文格式（Blockquote）

### 規則

1. 引文必須來自 `snippet` 或 `main_preview` 欄位，或使用者在上傳檔案中提供的判決全文原文。
2. 引文用 `> ` 標記，前後空行。
3. 引文結尾標示出處（盡可能精確到「理由欄第幾點」或「主文第幾項」；若 API 回傳無法定位，僅標「見同判決理由欄」）。
4. 引文若超過 100 字，必要時可節錄，並以「（中略）」標示省略處。
5. **絕對禁止將歸納後的白話整理放在 blockquote 中偽裝成原文**。

### 範例

```markdown
最高法院 108 年度台上字第 2027 號刑事判決指出：

> 主文、事實、理由之記載，附表一編號 2、5、6、7 中有關林益豪部分，
> 係經原審撤銷改判如附表一之一編號 1、2、3、4……

（見同判決理由欄）
```

## 五、卷證引用格式

### 完整格式（律師法庭用，最清楚）

```
{法院案號}卷第 {X} 頁（即起訴書第 {Y} 頁）
```

例：`115 金重訴 1 號卷第 187 頁（即起訴書第 175 頁）`

### 當僅有起訴書頁

```
起訴書第 {Y} 頁
```

### 當僅有偵查卷頁

```
{字號}卷 {卷號} 第 {Z} 頁
```

例：`113 偵 27269 卷 17 第 99 頁`

### 當使用者未提供頁碼

標示 `（頁碼待補）`。**絕對不可猜測頁碼。**

### 偏移公式使用注意

不同的卷宗可能有不同的「卷頁戳 + 原始頁」偏移。你（Claude）若沒看到實際 PDF，不能假設偏移；僅在使用者明確提供雙頁碼對照表時才可套用。

## 六、法條引用

### 格式

```
{法律名稱}第 {條} 條{第幾項}{第幾款}
```

範例：
- 刑事訴訟法第 159 條
- 刑事訴訟法第 159 條之 1 第 2 項
- 稅捐稽徵法第 41 條第 1 項
- 所得稅法第 11 條第 2 項

### 法律全名

第一次出現用全名，之後可用簡稱：

- 刑事訴訟法 → 刑訴法
- 稅捐稽徵法 → 稅徵法
- 組織犯罪防制條例 → 組織條例

---

## 七、API 失敗處理流程

1. 第一次呼叫失敗 → 等 2 秒後重試一次。
2. 重試仍失敗 → 在該判決旁標註「⚠ 驗證失敗（API 暫時不穩），建議律師手動至 dr-lawbot.com 確認」。
3. **絕對不可**因 API 失敗而略過驗證、直接引用判決。

---

## 八、資料涵蓋限制

- dr-lawbot 資料庫收錄民國 100 年 1 月 1 日起之判決。
- 89 年以前之最高行政法院判例（如「79 判 471」「74 判 1729」）不在資料庫內。
- 司法院判例制度已於 108 年廢止，但舊判例仍具參考價值；若使用者堅持引用，標註「舊判例，無法自動驗證」。

在交付大綱給使用者前，逐項確認：

- [ ] 所有 blockquote 引文都來自 API `main_preview` / `snippet` 或使用者提供的原文
- [ ] 沒有自行產生的卷頁號碼
- [ ] 法條引用完整（條、項、款）
- [ ] 沒有未驗證的舊判例被當作確實存在引用
- [ ] 最後附上判決清單（含 dr-lawbot 連結）
